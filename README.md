# 農業ドローン活用マップ（十勝／北海道）

国土交通省 Project LINKS の **無人航空機飛行計画データ（2025年度）** から、
**農林水産業目的のドローン飛行計画（申請）** を抽出し、北海道・十勝の市区町村別に可視化する静的 Web マップです。
JA・自治体・農業法人・農業 IT 関係者が、地域ごとの農業ドローン普及度を把握し、横展開や連携の検討材料にすることを想定しています。

> **本データは飛行「計画（申請）」であり、実際の飛行・実態を示すものではありません。**
> 紙資料のスキャン → 自動抽出によるデータのため、完全性・正確性は保証されません。座標は市区町村重心レベルに秘匿化されています。

## 主な集計結果（対象期間: 2024年7月〜2025年6月）

| 指標 | 件数 |
|---|---:|
| 走査した飛行計画 Feature（全用途・全国） | 2,971,260 |
| 農業目的の Feature | 1,229,417 |
| 農業目的の飛行計画（ID 単位・重複排除後） | 243,290 |
| うち北海道発 | 18,833（**全国1位**） |
| うち十勝管内発 | 1,690 |

- 北海道は農業目的の飛行計画件数で**全国1位**。
- 月別では融雪後の春〜初夏（5〜6月）に新規計画が集中し、明確な**散布シーズンの季節性**がみられます。

## 機能

- 市区町村別コロプレス地図（十勝管内／北海道全体の切替）
- 対象月フィルタ（全期間 累計／月次）
- 市区町村クリックで詳細（累計件数・機体種別・散布関連の飛行方法比率）
- 月別推移（北海道／十勝、または選択市区町村）
- 機体の種類・飛行方法（物件投下＝散布／目視外／夜間／30m未満）の内訳
- 全国 都道府県ランキング（文脈用、北海道を強調）

## ディレクトリ構成

```
scripts/
  download.sh          生データ取得（飛行計画 GeoJSON / 仕様書 / N03 境界）
  sources.tsv          飛行計画データのリソース定義（ファイル名・CKAN リソースID・対象月）
  preprocess.py        飛行計画 → 農業・北海道の市区町村別 軽量集計 JSON 生成
  build_boundaries.py  N03 行政区域 → 簡略化した市区町村境界 GeoJSON 生成
  verify_web.mjs       Playwright によるレンダリング検証
web/
  index.html, css/, js/, vendor/   静的サイト本体（Leaflet / Chart.js は同梱）
  data/                生成済みの軽量データ（このリポジトリに同梱）
    agri_municipalities.json       市区町村別 農業飛行計画 集計
    hokkaido_municipalities.geojson 簡略化境界
    prefecture_summary.json        都道府県別 件数
    meta.json                      生成メタ情報・出典・注記
```

`web/data/` は生成済みのためそのまま公開できます。生データ（`data/raw/`, 合計約 9GB）は再取得可能なので Git 管理外です。

## 再現手順

```bash
# 1. 生データ取得（約9GB。ネットワーク状況により時間がかかります）
bash scripts/download.sh

# 2. 集計 JSON 生成
pip install shapely
python3 scripts/preprocess.py

# 3. 市区町村境界 GeoJSON 生成
python3 scripts/build_boundaries.py

# 4. ローカル確認
npm run serve   # http://localhost:8000
```

## 前処理の方針（データの癖への対応）

- **規模**: 全件約297万 Feature。行ストリーム処理で必要な計画のみ抽出し、軽量集計に要約。
- **計数単位**: 1計画が複数の飛行エリア（ポリゴン）に分割されるため、**飛行計画ID 単位で重複排除**して「計画（申請）件数」を数える。
- **フィールド名の表記ゆれ**: 末尾スペース・全角括弧・CJK部首字を NFKC 正規化 + trim で吸収。
- **日時品質**: 飛行予定日時は品質が低いため集計に用いず、**計画IDの初出月**を新規計画の月として扱う。
- **包括申請ノイズ**: 業務目的フラグが極端に多い計画（しきい値8以上）を除外。
- **粒度**: 出発地（市区町村重心・秘匿化）粒度。個人・事業者を特定する二次加工は行わない。

## デプロイ

静的サイトのため GitHub Pages で公開できます（`.github/workflows/deploy.yml`、`web/` を配信）。
Pages を有効化のうえ `main` への push、または手動実行（workflow_dispatch）でデプロイされます。
